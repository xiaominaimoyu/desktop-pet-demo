# Phase 4 — 通用适配任务书

> **工期**: **10 个工作日** | **里程碑**: 任意 CLI 工具（无 hook 支持）也能通过文件/进程监控触发桌宠状态
> **版本标签**: `v0.1.0-rc2`（Phase 4 验收后打 tag）

---

## 一、阶段目标

实现**文件监控 + 进程监控 + 终端输出解析**三层降级方案，让桌宠在无 hook 的编码场景下也能工作。实现适配器自动发现与无缝切换。

> **工期调整说明**: 原计划 5 个工作日完成文件监控 + 进程监控 + 终端解析 + 适配器自动切换四大模块，工期严重不足。已调整为 10 个工作日，每个子模块分配 2-3 天以保证质量。

## 六、风险项

| 风险 | 等级 | 缓解措施 | 责任人 | 回退方案 |
|------|------|----------|--------|----------|
| **文件监控性能影响（大项目大量文件变更）** | **高** | 严格过滤目录（node_modules/target/.git/.DS_Store/Thumbs.db/\*.swp/\*.tmp）；实现事件风暴检测：1s 内 >100 事件时降级轮询模式 | 开发 | 关闭文件监控，仅保留进程监控 |
| 终端输出捕获在 Windows 上有权限限制 | 中 | 不捕获具体内容，只检测进程退出码 + 已知日志文件（如 `cargo build` 的 `target/` 输出） | 开发 | 仅使用退出码推断 |
| 进程匹配误报（非编码进程被识别为编码工具） | 低 | 使用进程命令行参数辅助判断（如 `node server.js` vs `node build.js`） | 开发 | 白名单模式：仅匹配用户配置的进程名 |
| `sysinfo` 在 Windows 上 CPU 占用偏高 | 低 | 增加轮询间隔到 5s；不使用 sysinfo 时回退到 WMI 查询 | 开发 | 仅在 CLI 工具运行时启用进程监控 |
| 中文终端输出漏检 | 中 | rustc 1.70+ 支持 `error:` / `错误:` 双语输出；使用正则同时匹配中英文 | 开发 | 只支持英文输出（强制 LANG=en） |
