## 1. 项目概述

### 1.1 项目结构

```
Desktop_Pet/
├── src/                      # 前端资源
│   ├── app.js               # 主应用逻辑
│   ├── bubble.js            # 气泡系统
│   ├── index.html           # 页面入口
│   ├── sprite-animator.js   # 精灵图动画
│   └── gif-animator.js      # GIF 动画
├── src-tauri/               # Tauri/Rust 后端
│   ├── src/
│   │   ├── main.rs         # 应用入口
│   │   ├── http.rs         # HTTP 服务 (Axum)
│   │   ├── mcp.rs          # MCP JSON-RPC 服务
│   │   ├── state.rs        # 状态管理
│   │   ├── movement.rs     # 运动系统
│   │   ├── adapter_*.rs   # CLI 适配器
│   │   ├── adapter_registry.rs  # 适配器注册表
│   │   └── process_detector.rs   # 进程检测
│   ├── Cargo.toml
│   └── tauri.conf.json
└── docs/                    # 项目文档
```

### 1.2 核心功能

- **适配器自动发现**: 自动检测 Claude Code、opencode、Copilot、TRAE 等 CLI 工具
- **运动模式切换**: 支持"自由漫游"和"编码跟随"两种模式
- **状态管理**: 完整的桌宠状态机（Idle、Running、Celebrating 等）
- **HTTP/MCP 服务**: 提供 HTTP hook 接口和 MCP JSON-RPC 接口

## 8. API 接口测试

### 8.1 HTTP API

#### 获取端口信息
```bash
curl http://localhost:9527/api/port
```

#### 获取适配器状态
```bash
curl http://localhost:9527/api/adapter/status
```

#### 适配器列表
```bash
curl http://localhost:9527/api/adapter/list
```

**响应示例**:
```json
{
  "active": "claude",
  "mode": "auto",
  "detected": ["claude", "opencode"]
}
```

#### 手动切换适配器
```bash
curl -X POST http://localhost:9527/api/adapter/switch \
  -H "Content-Type: application/json" \
  -d '{"id": "opencode"}'
```

#### 切换到自动模式
```bash
curl -X POST http://localhost:9527/api/adapter/switch \
  -H "Content-Type: application/json" \
  -d '{"mode": "auto"}'
```

### 8.2 Tauri 命令

#### 获取运动模式
```javascript
await invoke('get_movement_mode')
// 返回: "free_roam" 或 "coding_follow"
```

#### 设置运动模式
```javascript
await invoke('set_movement_mode', { newMode: 'coding_follow' })
```

#### 切换运动模式
```javascript
await invoke('toggle_movement_mode')
// 返回切换后的模式
```

### 8.3 MCP 接口

MCP 服务运行在 `localhost:9528`，使用 JSON-RPC 2.0 协议。

**可用工具**:
- `pet_show` - 显示桌宠
- `pet_hide` - 隐藏桌宠
- `pet_bubble` - 显示气泡消息
- `pet_state` - 获取当前状态
- `pet_input` - 发送用户输入

## 9. 常见问题排查

### 9.1 编译错误

#### 问题: `error: cannot find native static library 'webkit2gtk-4.1'`

**解决方案**:
```bash
# Ubuntu/Debian
sudo apt install libwebkit2gtk-4.1-dev

# Fedora
sudo dnf install webkit2gtk4.1-devel
```

#### 问题: `error: linker LINK.exe not found`

**解决方案**:
安装 Visual Studio Build Tools，选择"使用 C++ 的桌面开发"工作负载。

### 9.2 运行时错误

#### 问题: 端口被占用

```
Error: Could not find an available HTTP port (9527-9537)
```

**解决方案**:
1. 检查是否有其他程序占用了端口
2. 或等待端口释放后重试

```powershell
# 查看端口占用
netstat -ano | findstr "9527"
```

#### 问题: 窗口不显示

**可能原因**:
- 窗口被最小化到任务栏
- 窗口位置超出屏幕范围

**解决方案**:
修改 `src-tauri/tauri.conf.json` 中的窗口位置配置。

### 9.3 功能异常

#### 问题: 运动模式切换无响应

**排查步骤**:
1. 检查 `cargo test movement` 是否通过
2. 检查前端控制台是否有错误
3. 验证 Tauri 命令是否正确调用

#### 问题: 适配器未检测到

**排查步骤**:
1. 确认 CLI 工具已启动并正在运行
2. 检查进程名称是否符合检测规则
3. 查看日志中的进程扫描结果

### 9.4 日志获取

```bash
# 开发环境日志
cargo run 2>&1

# 查看详细日志（需要代码中已启用）
RUST_LOG=trace cargo run 2>&1
```
