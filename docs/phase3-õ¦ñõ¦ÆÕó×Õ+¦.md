## 四、逐日任务

### Day 1 — 移动系统基础：漫游 + 物理参数原型

| # | 任务 | 产出 | 估时 |
|---|------|------|------|
| 1.1 | 创建 `src/movement.js` — `MovementEngine` 类（参照 ameath 物理引擎移植） | 引擎骨架 | 2h |
| 1.2 | **物理参数原型验证**：写一个最小可跑原型，调试惯性/意图参数，确保在 JS 帧率下表现自然 | 参数原型 | **2h** |
| 1.3 | 实现随机目标点生成：屏幕范围内随机坐标，排除窗口自身区域 | 目标生成 | 1h |
| 1.4 | 实现惯性移动：每帧 `position += velocity * inertia`，`velocity += (target - position) * intention` | 惯性算法 | 1h |
| 1.5 | 实现边界处理：到达屏幕边缘反弹（速度取反）或瞬移到另一边 | 边界处理 | 1h |
| 1.6 | 前端集成：`app.js` 启动 `MovementEngine` → 桌宠开始在桌面漫游 | 联调 | 0.5h |

**Day 1 验收标准**: 桌宠在桌面上无目的漫游，碰壁反弹，运动自然。

> **关键风险 — 物理参数调优（高）**: ameath 的 Python 物理参数直接映射到 JS 可能存在帧率不一致问题（Python 60fps vs JS requestAnimationFrame 的 60fps 但调度机制不同）。Day 1 专门分配 2h 做参数原型验证，确保核心参数（惯性系数 0.95、意图系数 0.05）在 JS 环境下达到"自然感"。
