## 三、aemeath_withclaude-main 分析

### 3.1 技术架构
- **框架：** Tauri v1（`src-tauri/`）
- **前端：** 原生 HTML/CSS/JS
- **集成：** Claude Desktop 集成（CLAUDE.md, settings.json）
- **MCP 配置：** 通过 `docs/mcp.json` 定义 MCP 工具

### 3.2 关键档案
| 文件 | 内容 |
|------|------|
| `CLAUDE.md` | Claude 上下文指令 |
| `REDESIGN.md` | 重构设计方案 |
| `docs/API.md` | API 接口定义 |
| `docs/hooks.json` | Hook 事件配置 |
| `docs/mcp.json` | MCP 工具定义 |
| `src-tauri/tauri.conf.json` | Tauri 窗口配置 |
| `src/validation.json` | 数据验证规则 |

### 3.3 架构亮点
- **Rust + Tauri**：相比 Electron，Tauri 更轻量（包体小 60%+），内存占用更低
- **Claude 集成**：通过 `CLAUDE.md` 和 Hook 系统实现桌宠与 AI 编码助手的交互
- **MCP 协议支持**：定义了标准化的工具调用接口
- **前后端分离**：前端负责渲染和动画，Rust 后端负责状态管理和系统交互

### 3.4 可复用经验
- Tauri 透明窗口配置方案（`transparent: true`, `decorations: false`）
- 系统托盘集成方式（`tray.rs` 模式）
- Hook 事件数据流向：Agent → CLI → HTTP → 桌宠前端
- 状态管理与广播机制

## 四、架构决策对比

| 维度 | ameath (Electron) | aemeath (Tauri v1) | 本项目 (Tauri v2) |
|------|------------------|-------------------|-------------------|
| 窗口框架 | Electron BrowserWindow | Tauri Webview | Tauri Webview v2 |
| 后端 | Python | Rust | Rust |
| 通信 | 本地文件/进程 | HTTP / MCP | HTTP + MCP + Hook |
| 打包体积 | ~150MB+ | ~10MB | ~10MB |
| 系统API | Node.js | Rust + winapi | Rust + winapi + tauri2 API |
